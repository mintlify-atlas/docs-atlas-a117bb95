## Overview

Novel provides React hooks for common editor functionality. These hooks help manage state, storage, and editor interactions.

## useLocalStorage

A custom hook for managing localStorage with React state synchronization.

### Import

```typescript
import { useLocalStorage } from "novel";
```

### Type Signature

```typescript
function useLocalStorage<T>(
  key: string,
  initialValue: T
): [T, (value: T) => void]
```

**Source:** `~/workspace/source/packages/core/src/lib/hooks/use-local-storage.ts:3-7`

### Parameters

<ParamField path="key" type="string" required>
  The key to use for storing the value in localStorage.
  
  This should be unique to avoid conflicts with other data in localStorage.
</ParamField>

<ParamField path="initialValue" type="T" required>
  The initial value to use if no value exists in localStorage.
  
  This value is used on the first render and when localStorage is empty.
</ParamField>

### Returns

A tuple containing:

<ParamField path="[0]" type="T">
  The current stored value.
  
  Automatically synchronized with localStorage.
</ParamField>

<ParamField path="[1]" type="(value: T) => void">
  A function to update the stored value.
  
  Updates both React state and localStorage.
</ParamField>

### Usage Examples

#### Basic Usage

```tsx
import { useLocalStorage } from "novel";

function MyEditor() {
  const [content, setContent] = useLocalStorage("editor-content", "");
  
  return (
    <div>
      <textarea 
        value={content}
        onChange={(e) => setContent(e.target.value)}
      />
    </div>
  );
}
```

#### With TypeScript

```tsx
import { useLocalStorage } from "novel";
import { JSONContent } from "@tiptap/react";

function MyEditor() {
  const [content, setContent] = useLocalStorage<JSONContent>(
    "editor-content",
    { type: "doc", content: [] }
  );
  
  return (
    <Editor 
      defaultValue={content}
      onUpdate={(editor) => setContent(editor.getJSON())}
    />
  );
}
```

#### With Complex Objects

```tsx
import { useLocalStorage } from "novel";

interface EditorSettings {
  theme: "light" | "dark";
  fontSize: number;
  spellCheck: boolean;
}

function MyEditor() {
  const [settings, setSettings] = useLocalStorage<EditorSettings>(
    "editor-settings",
    {
      theme: "light",
      fontSize: 16,
      spellCheck: true,
    }
  );
  
  return (
    <div>
      <select 
        value={settings.theme}
        onChange={(e) => setSettings({ 
          ...settings, 
          theme: e.target.value as "light" | "dark" 
        })}
      >
        <option value="light">Light</option>
        <option value="dark">Dark</option>
      </select>
    </div>
  );
}
```

#### Clearing Storage

```tsx
import { useLocalStorage } from "novel";

function MyEditor() {
  const [content, setContent] = useLocalStorage("editor-content", "");
  
  const clearContent = () => {
    setContent("");
  };
  
  return (
    <div>
      <Editor defaultValue={content} />
      <button onClick={clearContent}>Clear Content</button>
    </div>
  );
}
```

## Implementation Details

### How it Works

1. **Initial Load**: On component mount, the hook reads from localStorage
2. **State Update**: When `setValue` is called, both React state and localStorage are updated
3. **JSON Serialization**: Values are automatically serialized/deserialized using JSON
4. **Hydration**: The stored value is loaded in a `useEffect` to avoid SSR issues

**Source:** `~/workspace/source/packages/core/src/lib/hooks/use-local-storage.ts:10-25`

### Storage Format

Values are stored as JSON strings:

```typescript
// Setting a value
window.localStorage.setItem(key, JSON.stringify(value));

// Getting a value
const item = window.localStorage.getItem(key);
if (item) {
  setStoredValue(JSON.parse(item));
}
```

### SSR Compatibility

<Warning>
  The hook uses `window.localStorage`, which is not available during server-side rendering. The hook safely handles this by using `initialValue` until the component hydrates on the client.
</Warning>

For Next.js or other SSR frameworks, the hook:
1. Returns `initialValue` during SSR
2. Loads from localStorage after hydration
3. Updates properly on the client

### Type Safety

The hook is fully generic and type-safe:

```typescript
// TypeScript infers the type from initialValue
const [count, setCount] = useLocalStorage("count", 0);
// count is number, setCount accepts number

const [user, setUser] = useLocalStorage("user", { name: "", age: 0 });
// user is { name: string; age: number }

// Or explicitly specify the type
const [data, setData] = useLocalStorage<MyType>("data", defaultValue);
```

## Limitations

### Storage Limits

- localStorage has a size limit (typically 5-10MB)
- Large documents may exceed this limit
- Consider using a database for large content

### Serialization

- Only JSON-serializable values are supported
- Functions, undefined, and symbols cannot be stored
- Date objects are converted to strings

### Browser Support

- Requires localStorage API (all modern browsers)
- Gracefully degrades if localStorage is unavailable
- May be blocked by privacy settings

## Best Practices

### Use Unique Keys

```tsx
// Good: Specific, namespaced key
const [content, setContent] = useLocalStorage("myapp-editor-content", "");

// Bad: Generic key that might conflict
const [content, setContent] = useLocalStorage("content", "");
```

### Handle Missing Storage

```tsx
function MyEditor() {
  const [content, setContent] = useLocalStorage("editor-content", "");
  
  useEffect(() => {
    // Check if storage is available
    try {
      localStorage.setItem("test", "test");
      localStorage.removeItem("test");
    } catch (e) {
      console.warn("localStorage not available");
    }
  }, []);
  
  return <Editor defaultValue={content} />;
}
```

### Combine with Editor

The Editor component uses this hook internally:

```typescript
// Inside Editor component
const [content, setContent] = useLocalStorage(storageKey, defaultValue);

const debouncedUpdates = useDebouncedCallback(async ({ editor }) => {
  const json = editor.getJSON();
  if (!disableLocalStorage) {
    setContent(json);
  }
}, debounceDuration);
```

## Related Hooks

While Novel currently only exports `useLocalStorage`, you can combine it with other React hooks:

- `useDebouncedCallback` (from use-debounce): Debounce storage updates
- `useEffect`: React to storage changes
- `useState`: Manage temporary state
- `useCallback`: Memoize storage update functions
