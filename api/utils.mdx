## Overview

Novel provides utility functions for common operations like styling, URL validation, and editor content manipulation.

## Styling Utilities

### cn

Combines class names using clsx and tailwind-merge.

```typescript
function cn(...inputs: ClassValue[]): string
```

**Source:** `~/workspace/source/packages/core/src/lib/utils.ts:4-6`

#### Parameters

<ParamField path="inputs" type="ClassValue[]" required>
  Class names to combine.
  
  Accepts strings, objects, arrays, and conditional classes. Automatically merges Tailwind classes intelligently.
</ParamField>

#### Returns

<ParamField path="className" type="string">
  The merged class name string.
  
  Duplicate and conflicting Tailwind classes are resolved automatically.
</ParamField>

#### Usage Examples

**Basic Usage:**

```typescript
import { cn } from "novel";

const className = cn("text-red-500", "bg-blue-500");
// Result: "text-red-500 bg-blue-500"
```

**Conditional Classes:**

```typescript
import { cn } from "novel";

const isActive = true;
const className = cn(
  "base-class",
  isActive && "active-class",
  !isActive && "inactive-class"
);
// Result: "base-class active-class"
```

**Merging Conflicting Tailwind Classes:**

```typescript
import { cn } from "novel";

const className = cn("px-4 py-2", "px-6");
// Result: "py-2 px-6" (px-4 is overridden by px-6)
```

**With Objects:**

```typescript
import { cn } from "novel";

const className = cn({
  "text-red-500": true,
  "bg-blue-500": false,
  "font-bold": true,
});
// Result: "text-red-500 font-bold"
```

**In React Components:**

```tsx
import { cn } from "novel";

function Button({ className, variant, ...props }) {
  return (
    <button
      className={cn(
        "px-4 py-2 rounded",
        variant === "primary" && "bg-blue-500 text-white",
        variant === "secondary" && "bg-gray-200 text-gray-900",
        className
      )}
      {...props}
    />
  );
}
```

## URL Utilities

### isValidUrl

Checks if a string is a valid URL.

```typescript
function isValidUrl(url: string): boolean
```

**Source:** `~/workspace/source/packages/core/src/lib/utils.ts:8-15`

#### Parameters

<ParamField path="url" type="string" required>
  The string to validate as a URL.
</ParamField>

#### Returns

<ParamField path="isValid" type="boolean">
  `true` if the string is a valid URL, `false` otherwise.
</ParamField>

#### Usage Examples

```typescript
import { isValidUrl } from "novel";

isValidUrl("https://example.com"); // true
isValidUrl("http://localhost:3000"); // true
isValidUrl("example.com"); // false (no protocol)
isValidUrl("not a url"); // false
isValidUrl(""); // false
```

**In Form Validation:**

```tsx
import { isValidUrl } from "novel";

function LinkForm() {
  const [url, setUrl] = useState("");
  const [error, setError] = useState("");
  
  const handleSubmit = () => {
    if (!isValidUrl(url)) {
      setError("Please enter a valid URL");
      return;
    }
    // Process valid URL
  };
  
  return (
    <form onSubmit={handleSubmit}>
      <input 
        value={url} 
        onChange={(e) => setUrl(e.target.value)}
        placeholder="https://example.com"
      />
      {error && <span>{error}</span>}
    </form>
  );
}
```

### getUrlFromString

Converts a string to a valid URL, adding protocol if needed.

```typescript
function getUrlFromString(str: string): string | null
```

**Source:** `~/workspace/source/packages/core/src/lib/utils.ts:17-27`

#### Parameters

<ParamField path="str" type="string" required>
  The string to convert to a URL.
  
  Can be a full URL or a domain name.
</ParamField>

#### Returns

<ParamField path="url" type="string | null">
  A valid URL string with protocol, or `null` if the string cannot be converted.
  
  Automatically adds `https://` if the string looks like a domain.
</ParamField>

#### Usage Examples

```typescript
import { getUrlFromString } from "novel";

getUrlFromString("https://example.com"); 
// "https://example.com"

getUrlFromString("example.com"); 
// "https://example.com"

getUrlFromString("github.com/user/repo"); 
// "https://github.com/user/repo"

getUrlFromString("not a url with spaces"); 
// null

getUrlFromString("just-text"); 
// null (no dot, not a domain)
```

**Smart URL Input:**

```tsx
import { getUrlFromString } from "novel";

function LinkInput() {
  const [input, setInput] = useState("");
  
  const handleSubmit = () => {
    const url = getUrlFromString(input);
    if (url) {
      // Use the normalized URL
      window.open(url, "_blank");
    } else {
      alert("Invalid URL");
    }
  };
  
  return (
    <div>
      <input 
        value={input}
        onChange={(e) => setInput(e.target.value)}
        placeholder="Enter URL or domain"
      />
      <button onClick={handleSubmit}>Go</button>
    </div>
  );
}
```

**In Editor Link Extension:**

```typescript
import { getUrlFromString } from "novel";

function insertLink(editor, text) {
  const url = getUrlFromString(text);
  if (url) {
    editor
      .chain()
      .focus()
      .setLink({ href: url })
      .run();
  }
}
```

## Editor Utilities

### getPrevText

Extracts previous text from the editor before the current cursor position.

```typescript
function getPrevText(
  editor: Editor,
  options: { chars: number; offset?: number }
): string
```

**Source:** `~/workspace/source/packages/core/src/lib/editor.ts:3-21`

#### Parameters

<ParamField path="editor" type="Editor" required>
  The Tiptap editor instance.
</ParamField>

<ParamField path="options.chars" type="number" required>
  The maximum number of characters to extract.
  
  Extracts up to this many characters before the cursor.
</ParamField>

<ParamField path="options.offset" type="number" default="0">
  Number of characters to offset from the cursor position.
  
  Useful for excluding trigger characters like `++`.
</ParamField>

#### Returns

<ParamField path="text" type="string">
  The extracted text from the editor.
  
  Text is extracted using `textBetween` with newline preservation.
</ParamField>

#### Usage Examples

**Basic Usage:**

```typescript
import { getPrevText } from "novel";

const text = getPrevText(editor, { chars: 100 });
console.log(text); // Last 100 characters before cursor
```

**With Offset:**

```typescript
import { getPrevText } from "novel";

// Exclude the last 2 characters (e.g., "++" trigger)
const text = getPrevText(editor, { chars: 5000, offset: 2 });
```

**For AI Completion:**

```typescript
import { getPrevText } from "novel";
import { useCompletion } from "ai/react";

function useAICompletion(editor) {
  const { complete } = useCompletion({
    api: "/api/generate",
  });
  
  const triggerCompletion = () => {
    const context = getPrevText(editor, { 
      chars: 5000,
      offset: 1 
    });
    complete(context);
  };
  
  return { triggerCompletion };
}
```

**In Slash Command:**

```typescript
import { getPrevText } from "novel";

// From slash-command.tsx
const handleContinueWriting = () => {
  const context = getPrevText(editor, {
    chars: 5000,
    offset: 1,
  });
  complete(context);
};
```

**Source reference:** `~/workspace/source/packages/core/src/ui/editor/extensions/slash-command.tsx:304-308`

## Implementation Details

### cn (Class Name Utility)

Uses:
- **clsx**: Conditional class name composition
- **tailwind-merge**: Intelligent Tailwind class merging

This combination ensures:
- Conditional classes work correctly
- Tailwind utility conflicts are resolved
- Later classes override earlier ones
- No duplicate classes in output

### URL Utilities

Both URL utilities use the native `URL` constructor for validation:
- Throws for invalid URLs
- Handles all URL edge cases
- Supports all valid URL protocols
- Works with IP addresses and ports

### getPrevText

Uses ProseMirror's `textBetween` method:
- Preserves newlines in text
- Efficiently extracts text ranges
- Handles document boundaries
- Works with all node types

<Warning>
  The current implementation uses `textBetween` which extracts plain text. Formatting is not preserved. For markdown extraction, you would need to use `editor.storage.markdown.getMarkdown()` (requires Markdown extension).
</Warning>

## Best Practices

### Using cn()

```typescript
// Good: Clear separation of base and conditional classes
const className = cn(
  "base-class another-base",
  condition && "conditional-class",
  otherCondition && "other-conditional"
);

// Bad: Hard to read, mixing concerns
const className = cn("base-class", condition && "conditional-class", "another-base");
```

### URL Validation

```typescript
// Good: Use getUrlFromString for user input
const url = getUrlFromString(userInput);
if (url) {
  // Use normalized URL
}

// Bad: Assume user input is formatted correctly
if (isValidUrl(userInput)) {
  // userInput might be missing protocol
}
```

### Text Extraction

```typescript
// Good: Use appropriate character limit
const context = getPrevText(editor, { chars: 5000 });

// Bad: Extracting entire document
const context = getPrevText(editor, { chars: 1000000 });
```
