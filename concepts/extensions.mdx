## Overview

Novel's power comes from its extension system. Extensions are modular pieces of functionality that add features to the editor—from basic text formatting to complex behaviors like slash commands and AI completions.

<Info>
Novel is built on [Tiptap](https://tiptap.dev/), which has a robust extension ecosystem. Novel provides a curated set of default extensions optimized for modern content editing.
</Info>

## Default Extensions

Novel includes 15+ built-in extensions (defined internally in `packages/core/src/ui/editor/extensions/index.tsx`) that work together automatically:

```tsx
import { Editor } from "novel";

// All default extensions are automatically included
<Editor />

// Add your custom extensions on top of the defaults
<Editor extensions={[MyCustomExtension]} />
```

<Note>
The `defaultExtensions` array is not exported from the package. Novel automatically includes all default extensions, and any extensions you pass via the `extensions` prop are added on top of them.
</Note>

```tsx
    HTMLAttributes: {
      class: "novel-rounded-sm novel-bg-stone-100 novel-p-5 novel-font-mono novel-font-medium novel-text-stone-800",
    },
  },
  // ... more configurations
})
```

**What's included in StarterKit:**
- Bold, Italic, Strike formatting
- Headings (H1-H6)
- Paragraphs
- Bullet and ordered lists
- Code blocks and inline code
- Blockquotes
- Hard breaks
- Horizontal rules (disabled, see below)
- History (undo/redo)
- Drop cursor and gap cursor

<Note>
Novel disables StarterKit's default `horizontalRule` and `gapcursor` to provide custom implementations.
</Note>

#### HorizontalRule (Extended)

Novel extends the default HorizontalRule to fix a Tiptap bug:

```tsx
// From extensions/index.tsx:64-88
HorizontalRule.extend({
  addInputRules() {
    return [
      new InputRule({
        find: /^(?:---|—-|___\s|\*\*\*\s)$/,
        handler: ({ state, range }) => {
          const attributes = {};
          const { tr } = state;
          const start = range.from;
          let end = range.to;
          
          tr.insert(start - 1, this.type.create(attributes))
            .delete(tr.mapping.map(start), tr.mapping.map(end));
        },
      }),
    ];
  },
}).configure({
  HTMLAttributes: {
    class: "novel-mt-4 novel-mb-6 novel-border-t novel-border-stone-300",
  },
})
```

**Why the extension?** This fixes [issue #3859](https://github.com/ueberdosis/tiptap/pull/3859#issuecomment-1536799740) where typing `---`, `___`, or `***` didn't properly insert horizontal rules.

**Why two image extensions?**
- **TiptapImage (extended)**: Adds `UploadImagesPlugin` for handling image uploads, supports base64
- **UpdatedImage**: Custom extension for enhanced image capabilities

Both render with rounded corners and borders for consistent styling.

### Placeholder Extension

**Smart placeholders:**

### Custom Novel Extensions

Novel includes three custom-built extensions:

#### 1. SlashCommand

```tsx
// From extensions/index.tsx:119
SlashCommand
```

Pressing `/` opens a command menu for inserting blocks and triggering AI features. See `extensions/slash-command.tsx:48-76` for implementation.

**Available commands:**
- Continue writing (AI)
- Ask a question AI
- Headings (1-3)
- Bullet/numbered lists
- To-do lists
- Quotes
- Code blocks
- Images

<Note>
SlashCommand uses Tiptap's `Suggestion` utility under the hood to create a pop-up menu with keyboard navigation.
</Note>

#### 2. CustomKeymap

```tsx
// From extensions/index.tsx:142
CustomKeymap
```

Enhances the default Cmd/Ctrl+A behavior:

```tsx
// From extensions/custom-keymap.ts:36-52
addKeyboardShortcuts() {
  return {
    "Mod-a": ({ editor }) => {
      const { state } = editor;
      const { tr } = state;
      const startSelectionPos = tr.selection.from;
      const endSelectionPos = tr.selection.to;
      const startNodePos = tr.selection.$from.start();
      const endNodePos = tr.selection.$to.end();
      
      const isCurrentTextSelectionNotExtendedToNodeBoundaries =
        startSelectionPos > startNodePos || endSelectionPos < endNodePos;
      
      if (isCurrentTextSelectionNotExtendedToNodeBoundaries) {
        editor.chain().selectTextWithinNodeBoundaries().run();
        return true;
      }
      return false;
    },
  };
}
```

**Why this matters:** First Cmd+A selects text within the current node (e.g., paragraph), second Cmd+A selects all. This matches user expectations from apps like Notion.

#### 3. DragAndDrop

```tsx
// From extensions/index.tsx:143
DragAndDrop
```

Enables drag-and-drop reordering of blocks.

## Adding Custom Extensions

You can extend Novel with your own or third-party extensions:

```tsx
import { Editor } from "novel";
import { Mathematics } from "@tiptap-pro/extension-mathematics";
import CustomExtension from "./my-extension";

function MyEditor() {
  return (
    <Editor
      extensions={[
        Mathematics.configure({
          // config
        }),
        CustomExtension,
      ]}
    />
  );
}
```

<Warning>
Extensions are merged internally as `[...defaultExtensions, ...extensions]`. Your custom extensions are added after the defaults. If you need to override a default extension's behavior, create a custom extension with the same name and higher priority - Tiptap will use your version instead.
</Warning>

### Overriding Default Extensions

Since `defaultExtensions` is not exported, you cannot filter them directly. Instead, override by priority:

```tsx
import { Editor } from "novel";
import Link from "@tiptap/extension-link";

// Create a custom link extension with higher priority
const CustomLink = Link.extend({
  priority: 1000, // Higher than default (100)
  // Your customizations
});

function MyEditor() {
  return (
    <Editor
      extensions={[CustomLink]}
    />
  );
}
```

Tiptap will use the extension with the highest priority when multiple extensions have the same name.

## Extension Configuration Deep Dive

### How Extensions Are Merged

Internally, Novel merges extensions (from `index.tsx:99`):

```tsx
const editor = useEditor({
  extensions: [...defaultExtensions, ...extensions],
  // ...
});
```

Novel's default extensions are loaded first, then your custom extensions are added. Tiptap processes them in order, with priority determining precedence for extensions with duplicate names.

### Extension Priority

Extensions can define priority to control ordering:

```tsx
const MyExtension = Extension.create({
  name: 'myExtension',
  priority: 1000, // Higher = processed first
  // ...
});
```

<Info>
Most default extensions don't set explicit priority, so they use Tiptap's default (100).
</Info>

## Extension Capabilities

Tiptap extensions can:

- **Add nodes** (block-level content like paragraphs, headings)
- **Add marks** (inline formatting like bold, links)
- **Add commands** (programmatic actions)
- **Add keyboard shortcuts**
- **Add input rules** (auto-formatting as you type)
- **Add paste rules** (transform pasted content)
- **Add ProseMirror plugins** (low-level editor behavior)
