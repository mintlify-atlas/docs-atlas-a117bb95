This guide will help you integrate the Novel editor into your React application and start editing.

## Basic usage

The simplest way to use Novel is to import the `Editor` component and render it in your app:

```jsx App.jsx
import { Editor } from "novel";

export default function App() {
  return <Editor />;
}
```

That's it! The editor will render with default styling and functionality.

## Using the editor

Once the editor is rendered, you can:

<Steps>

<Step title="Type content">
  Click into the editor and start typing. The placeholder text will guide you:
  
  - Press `/` for commands
  - Type `++` for AI autocomplete (if configured)
</Step>

<Step title="Use slash commands">
  Type `/` to open the command menu and access:
  
  - Headings (H1, H2, H3)
  - Lists (bullet, numbered, task)
  - Code blocks
  - Images
  - Horizontal rules
  - And more
</Step>

<Step title="Format with the bubble menu">
  Select any text to reveal the bubble menu with formatting options:
  
  - Bold, italic, underline
  - Text color and highlights
  - Code formatting
  - Links
</Step>

<Step title="Trigger AI completions">
  Type `++` anywhere to trigger AI-powered autocompletions based on your context (requires API endpoint setup).
</Step>

</Steps>

## Customizing the editor

Novel provides several props to customize behavior and appearance:

```jsx App.jsx
import { Editor } from "novel";

export default function App() {
  return (
    <Editor
      className="min-h-[300px] w-full max-w-4xl"
      defaultValue={{
        type: "doc",
        content: [
          {
            type: "heading",
            attrs: { level: 1 },
            content: [{ type: "text", text: "Welcome to Novel" }],
          },
        ],
      }}
      onUpdate={(editor) => {
        console.log("Content updated:", editor?.getJSON());
      }}
      storageKey="my-novel-content"
    />
  );
}
```

### Common props

- `className` - Custom CSS classes for the editor container
- `defaultValue` - Initial content as JSONContent or string
- `onUpdate` - Callback fired on every content change
- `onDebouncedUpdate` - Debounced update callback (default: 750ms)
- `storageKey` - Local storage key for persisting content
- `disableLocalStorage` - Disable automatic local storage persistence

<Tip>
  The editor automatically saves content to local storage by default. Use a unique `storageKey` for each editor instance to prevent conflicts.
</Tip>

## Accessing editor content

You can access the editor's content through the `onUpdate` and `onDebouncedUpdate` callbacks:

```jsx App.jsx
import { Editor } from "novel";
import { useState } from "react";

export default function App() {
  const [content, setContent] = useState(null);

  return (
    <>
      <Editor
        onDebouncedUpdate={(editor) => {
          setContent(editor?.getJSON());
        }}
        debounceDuration={1000}
      />
      
      <pre>
        {JSON.stringify(content, null, 2)}
      </pre>
    </>
  );
}
```

<Note>
  Use `onDebouncedUpdate` instead of `onUpdate` when saving to a database or API to avoid excessive calls. The default debounce duration is 750ms.
</Note>

## Adding custom extensions

Novel is built on Tiptap, so you can extend it with custom Tiptap extensions:

```jsx App.jsx
import { Editor } from "novel";
import { Color } from "@tiptap/extension-color";
import { TextStyle } from "@tiptap/extension-text-style";

export default function App() {
  return (
    <Editor
      extensions={[
        // Add custom extensions here
        // They will be merged with Novel's default extensions
      ]}
    />
  );
}
```

<Warning>
  Novel already includes many common Tiptap extensions like StarterKit, Link, Image, TaskList, and more. Check the default extensions in the [source code](https://github.com/steven-tey/novel/blob/main/packages/core/src/ui/editor/extensions/index.tsx) before adding your own to avoid conflicts.
</Warning>

## Customizing editor props

You can pass additional props to the underlying Tiptap editor using the `editorProps` option:

```jsx App.jsx
import { Editor } from "novel";

export default function App() {
  return (
    <Editor
      editorProps={{
        attributes: {
          class: "prose prose-lg focus:outline-none max-w-full",
        },
      }}
    />
  );
}
```

These props are merged with Novel's default editor props, which handle image uploads, paste events, and keyboard shortcuts.
